# ADR-0004: 파일 기반 이슈 추적

- **상태:** 승인됨
- **날짜:** 2026-03-14

## 맥락

버그, 기능, 기술 결정을 추적할 시스템이 필요했습니다. 호스팅 플랫폼 (GitHub Issues, Linear)과 레포지토리 내 파일 기반 추적 중 선택.

## 검토한 옵션

| 옵션 | 장점 | 단점 |
|------|------|------|
| GitHub Issues | 웹 UI, PR/커밋 연동, 레이블, 마일스톤 | 온라인 전용, git clone에 미포함, 벤더 종속 |
| Linear | 빠른 UI, 우수한 워크플로우 | 팀은 유료, 외부 의존성 |
| 파일 기반 (docs/issues/) | 오프라인, 코드와 함께 버전 관리, AI 에이전트가 직접 읽기 가능, 이식성 | UI 없음, 수동 검색/필터 |

## 결정

`docs/issues/`에 구조화된 마크다운 템플릿을 사용한 파일 기반 이슈 추적.

## 근거

- 오프라인 우선이 로컬 우선 앱 철학과 일치
- AI 에이전트가 이슈 파일을 glob하고 읽어 즉시 컨텍스트 확보 가능
- 이슈가 수정하는 코드와 함께 버전 관리됨
- 필요 시 나중에 GitHub Issues로 이전 가능
- 템플릿에 기술 분석, 보안 고려, 테스트 검증 포함

## 결과

- 이슈 브라우징을 위한 웹 UI 없음 — 파일 탐색기 / grep에 의존
- 이슈 파일 생성에 규율 유지 필요
- 커밋에서 이슈 참조는 `#ISSUE-NNN` 형식 사용
