# ADR-0007: Drift를 로컬 데이터베이스로 (Isar 교체)

- **상태:** 승인됨
- **날짜:** 2026-03-15
- **결정자:** 사용자, Claude

## 맥락

Isar가 원래 로컬 데이터베이스로 선택되었습니다 (ADR-0001). Phase 1 구현 중 다음이 발견되었습니다:

1. **Isar가 방치됨** — 더 이상 활발히 유지보수되지 않음
2. **의존성 충돌** — `isar_generator`가 `build_runner`와 충돌하여 코드 생성 차단

타입 안전 쿼리, 코드 생성, 향후 Supabase 마이그레이션을 위한 repository 패턴을 지원하는 대체재가 필요했습니다.

## 결정

**Drift** (SQLite 기반)를 로컬 데이터베이스로 사용.

## 근거

- **활발한 유지보수** — 정기적 릴리스, 반응적인 메인테이너
- **타입 안전 쿼리** — Dart 코드 생성을 통한 컴파일 타임 검증 SQL
- **리액티브 스트림** — `watch()` 쿼리가 Riverpod과 자연스럽게 통합
- **마이그레이션 지원** — 프로덕션 업그레이드를 위한 구조화된 스키마 버저닝
- **SQLite 기반** — 검증된 저장 엔진, 넓은 플랫폼 지원
- **Isolate 지원** — 무거운 쿼리를 메인 스레드 밖에서 실행 가능

## 결과

- 코드 생성에 `build_runner` 필요 (Isar도 마찬가지였을 것)
- SQL 지식이 도움이 되지만 필수는 아님 (Drift가 Dart 쿼리 API 제공)
- 데이터베이스 파일이 표준 SQLite 파일 (외부 도구로 검사 가능)
- `drift_flutter` 패키지가 플랫폼별 SQLite 번들링 처리
