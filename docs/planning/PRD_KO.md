# Taptime - 제품 요구사항 문서

> **버전:** 1.1
> **최종 업데이트:** 2026-03-14
> **상태:** 초안

## 1. 개요

**Taptime**은 커스터마이즈 가능한 프리셋과 내장 뽀모도로 타이머를 사용하여 시간을 손쉽게 기록할 수 있는 시간 관리 앱입니다. 사용자가 프리셋을 탭하면 타이머가 시작되고, 완료된 세션은 자동으로 기록되어 통계 및 향후 Google Calendar 동기화에 활용됩니다.

### 1.1 비전

시간 추적을 한 번의 탭만큼 간단하게 — 사용자가 실제로 꾸준히 사용하도록.

### 1.2 대상 사용자

- **주요:** 시간 사용을 추적하고 개선하고 싶은 개인 (학생, 프리랜서, 자기학습자, 생산성 애호가)

### 1.3 플랫폼

- **Flutter** (iOS + Android, 향후 웹 지원 가능)
- **로컬 우선** 저장, 향후 클라우드 동기화 기능 추가 가능

## 2. 핵심 개념

### 2.1 프리셋

사용자가 정의하는 시간 활동 템플릿.

| 필드 | 설명 | 예시 |
|------|------|------|
| `name` | 활동 이름 | "공부", "운동" |
| `duration` | 기본 타이머 시간 (분) | 25, 60 |
| `icon` | 시각적 식별자 | book, dumbbell |
| `color` | 카테고리 색상 | `#4A90D9` |
| `dailyGoal` | 선택적 일일 목표 (분) | 120 |
| `weeklyGoal` | 선택적 주간 목표 (분) | 600 |

### 2.2 세션

타이머가 실행될 때 생성되는 기록된 시간 블록.

| 필드 | 설명 |
|------|------|
| `presetId` | 연결된 프리셋 |
| `startedAt` | 타이머 시작 타임스탬프 |
| `endedAt` | 타이머 정지/완료 타임스탬프 |
| `durationMinutes` | 반올림된 소요 시간 (분) |
| `durationSeconds` | 정밀 소요 시간 (초, 내부용) |
| `status` | `completed` / `stopped` |
| `memo` | 선택적 한 줄 메모 (세션 후) |

### 2.3 일일 목표

프리셋별 또는 전체 일일/주간 시간 목표. 기록된 세션에서 진행률을 계산합니다.

## 3. 기능 사양

### 3.1 프리셋 관리

- **생성** — 이름, 시간, 아이콘, 색상으로 커스텀 프리셋 생성
- **수정** — 기존 프리셋 편집
- **삭제** — 프리셋 삭제 (세션은 히스토리에 유지)
- **재정렬** — 홈 화면에서 드래그 앤 드롭으로 재정렬
- **기본 프리셋** — 첫 실행 시 제공 (예: 공부 25분, 운동 30분, 독서 20분)

### 3.2 타이머

- 프리셋을 탭하면 **카운트다운 뽀모도로 타이머** 시작
- 타이머 표시: 남은 시간, 프리셋 이름, 진행 링/바
- 컨트롤: **일시정지**, **재개**, **정지**
- 완료 시: 사운드/진동 알림, 자동 세션 저장
- 수동 정지 시: 실제 경과 시간으로 세션 저장
- 선택: 완료 후 설정 가능한 휴식 타이머 (5분 짧은 / 15분 긴)
- 백그라운드 타이머 지원 (앱 최소화 시에도 계속)

### 3.3 세션 기록

- 타이머 완료 또는 수동 정지 시 세션 자동 저장
- 내부적으로 초 단위 정밀도로 저장, 분 단위로 표시
- 선택적 세션 후 메모 (한 줄 회고)
- 수동 세션 입력 (과거 활동을 직접 추가)
- 기록된 세션 편집/삭제

### 3.4 통계 & 시각화

- **일간 뷰:** 세션 타임라인, 프리셋별 총 시간
- **주간 뷰:** 일별 총 시간 막대 차트, 카테고리 분류
- **월간 뷰:** 히트맵 캘린더 (강도 = 총 시간), 트렌드
- **목표 진행률:** 프리셋별 일일/주간 목표 달성 비율
- **스트릭:** 목표 달성 연속 일수

### 3.5 목표 시스템

- 프리셋별 일일 목표 시간 설정 (예: 공부: 2시간/일)
- 프리셋별 주간 목표 시간 설정
- 전체 일일 목표 설정 (모든 프리셋 합산)
- 홈 화면에 시각적 진행률 표시
- 연속 목표 달성 스트릭 카운터

### 3.6 히트맵 & 스트릭

- **히트맵:** 일일 활동 강도를 보여주는 GitHub 스타일 기여 캘린더
- **스트릭:** 일일 목표 달성 연속 일수, 홈 화면에 표시
- 스트릭 마일스톤 (7, 30, 100일) 시각적 축하

### 3.7 데이터 내보내기/가져오기

- 모든 데이터를 JSON으로 내보내기 (백업/복원)
- JSON에서 가져오기 (기기 이전)
- MVP 범위 — 클라우드 백업 전 최소한의 안전망

### 3.8 동기부여 이펙트 (MVP 이후)

- 타이머 완료 시 완료 애니메이션 (폭죽, 글로우 등)
- 누적 시각화 (예: 블록 쌓기, 정원 가꾸기)
- 업적 뱃지

### 3.9 Google Calendar 연동 (MVP 이후)

- 세션을 캘린더 이벤트로 내보내기
- 아키텍처: 캘린더 어댑터 인터페이스가 있는 repository 패턴
- `flutter_secure_storage`를 사용한 클라이언트 사이드 OAuth2 토큰 관리
- 백엔드 불필요 — 클라이언트에서 직접 Google API 호출

### 3.10 클라우드 백업 (MVP 이후)

- 클라우드 백업/복원을 위한 Supabase 연동
- 소셜 로그인 (Google/Apple Sign-In) — 커스텀 계정 시스템 없음
- Auth 인터페이스는 정의하되 MVP에서는 `NoAuth` (패스스루)로 구현
- 목적: 기기 변경 시 데이터 보존, 멀티 디바이스 동기화가 아님

## 4. 디자인 원칙

### 4.1 비주얼

- **최소 팔레트:** 최대 2-3가지 색상 (주색, 강조색, 중립색)
- **타이포그래피 중심:** 깔끔하고 읽기 좋은, 넉넉한 여백
- **다크 모드** 처음부터 지원 (Flutter ThemeData)

### 4.2 UX

- 홈 화면 = 프리셋 그리드. 한 번 탭으로 시작.
- 어떤 기능이든 최대 2번 탭으로 도달
- 온보딩 장벽 없음 — 기본 프리셋으로 즉시 사용 가능
- 타이머 시작/정지 시 햅틱 피드백

### 4.3 제안 컬러 팔레트

| 역할 | 라이트 모드 | 다크 모드 |
|------|-----------|----------|
| 주색 | `#1A1A2E` (딥 네이비) | `#E8E8F0` (소프트 화이트) |
| 강조색 | `#E94560` (코랄 레드) | `#E94560` (코랄 레드) |
| 배경 | `#FAFAFA` (오프 화이트) | `#16213E` (다크 블루) |
