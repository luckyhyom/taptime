# 서브에이전트 & 플랜 모드 — 사용 시점

> 작성: 2026-03-15

## 서브에이전트 유형

| 에이전트 | 하는 일 | 사용 시점 |
|---------|---------|----------|
| **Explore** | 빠른 코드베이스 검색 (파일, 키워드) | 코드베이스가 커진 후 (Phase 2+). 작은 코드베이스에는 Glob/Grep이 더 빠름 |
| **Plan** | 아키텍처 설계, 구현 전략 | 여러 파일에 걸친 복잡한 구현 전 |
| **docs-researcher** (Context7) | 라이브러리 문서 조회 | 메인 컨텍스트 오염 없이 라이브러리 문서 (Drift, Riverpod 등) 참조 시 |
| **general-purpose** | 다단계 복잡한 태스크 | 독립적 조사, 다중 파일 탐색 |

### 팁

- 작은 코드베이스에서는 직접 도구 사용 (Glob/Grep/Read)이 서브에이전트 생성보다 빠름
- 서브에이전트는 독립적인 태스크를 병렬화할 때 가장 유용
- docs-researcher는 라이브러리 문서를 별도 컨텍스트에서 가져와 메인 컨텍스트 윈도우 토큰 절약

## 플랜 모드

### 무엇인가

구현 전 설계 우선 모드. 코드 수정/생성이 비활성화됨 — 읽기와 검색만 가능. "건물 짓기 전 설계도 그리기"와 같음.

### 사용 시점

1. **여러 파일에 걸친 대규모 기능 전** — 예: 폴더 구조 + 테마 + 라우터 + DB 한꺼번에
2. **아키텍처 결정이 필요할 때** — 예: DB 테이블 구조, 프로바이더 와이어링 전략
3. **파일 의존성이 복잡할 때** — 순서를 잘못하면 빌드가 깨짐

### 사용하지 않을 때

- 단일 파일 수정 (의존성 교체, 문서 업데이트)
- 단순한 태스크 (파일 생성, 설정 변경)
- PLAN.md에 이미 상세히 기술된 단순 태스크

### 주의사항

- **남용 금지** — 단순 태스크에 플랜 모드는 시간 낭비
- **플랜 ≠ 실행 보장** — 구현 중 변경이 생길 수 있음; 경직되지 말 것
- **토큰 소비** — 플랜 모드 탐색도 토큰을 소비하지만, 잘못된 방향으로 구현 후 되돌리는 것보다 훨씬 저렴
