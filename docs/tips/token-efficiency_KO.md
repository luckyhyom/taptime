# 토큰 효율 팁

> AI 토큰 사용량을 최소화하기 위한 문서 및 대화 관리법.

## 작성 언어

- 영어는 동일 내용 대비 한국어보다 ~1.5-2배 적은 토큰 사용
- AI 출력 토큰은 입력 토큰보다 5배 비용 (Claude API 가격)
- 베스트 프랙티스: 문서는 영어로, 대화는 사용자의 선호 언어로

## 읽기 빈도에 따른 파일 배치

| 위치 | 로드 시점 | 여기에 넣을 것 |
|------|----------|-------------|
| `.claude/rules/` | 매 세션 (자동) | 항상 따라야 하는 규칙 |
| `CLAUDE.md` | 매 세션 (자동) | 워크플로우 규칙, 200줄 미만 유지 |
| `docs/references/` | 필요 시 (수동 읽기) | 조사 자료, 드물게 참조 |
| `docs/adr/` | 필요 시 (수동 읽기) | 아키텍처 결정, 관련 시 읽기 |

- `.claude/rules/`에 대용량 조사 문서 **넣지 말 것** — 매 세션 로드되어 토큰 낭비
- `.claude/rules/`에는 간결하고 실행 가능한 규칙을 **넣을 것**

## 검색 vs 읽기

| 작업 | 토큰 비용 |
|------|----------|
| 파일 쓰기 | 최소 (도구 호출 1회) |
| 파일 읽기 | 파일 크기에 비례 |
| 웹 검색 + 파싱 + 요약 | **가장 비쌈** |

- 조사 결과를 `docs/references/`에 저장 — 저장된 파일 읽기가 재검색보다 훨씬 저렴
- 참고 파일은 주제에 집중 — 대용량 조사는 별도 주제 파일로 분리

## 대화 스타일

- 사용자 한국어 + AI 한국어 응답: 표준 비용
- 사용자 한국어 + AI 영어 응답: **가장 비용 효율적** (출력이 더 길고 비쌈, 영어 출력이 적은 토큰)
- 현실적 선택: 가장 편한 언어 사용 — 사용량 한도에 부딪힐 때만 최적화
