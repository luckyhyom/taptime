# 인덱싱과 메모리

INDEX.md와 MEMORY.md의 작동 방식, 에이전트가 프로젝트 문서를 검색하는 방법.

## INDEX.md vs MEMORY.md

| | MEMORY.md | INDEX.md |
|---|---|---|
| 목적 | 세션 간 메모리 (사용자 정보, 피드백, 프로젝트 상태) | 프로젝트 문서 맵 |
| 자동 로드 | 예, 매 대화에 주입 | 아니오, 에이전트가 수동으로 읽음 |
| 크기 한도 | 200줄 (하드 리밋, 초과 시 잘림) | 하드 리밋 없음 (짧을수록 좋음) |
| 겹침 | 없음 — 용도가 다름 | 없음 |

## 에이전트가 문서를 찾는 방법

```
CLAUDE.md (자동 주입)
  → "docs/INDEX.md 참조" 언급
    → 에이전트가 INDEX.md 읽음
      → 필요한 문서를 찾아서 읽음
```

이것은 자동이 아닌 수동 과정. 에이전트 판단에 의존.

## 인덱싱이 실패할 때

- 문서가 INDEX.md에 등록 안 됨 → 에이전트가 존재를 모름
- CLAUDE.md가 INDEX.md를 언급 안 함 → 에이전트가 아예 건너뛸 수 있음
- 서브에이전트 (Agent 도구)가 CLAUDE.md를 받지 못할 수 있음 → 메인 에이전트가 프롬프트에 경로를 명시해야 함

## 현재 안전장치

CLAUDE.md가 INDEX.md를 3번 참조 (3, 31, 79번째 줄), 메인 에이전트가 놓칠 가능성이 매우 낮음.
