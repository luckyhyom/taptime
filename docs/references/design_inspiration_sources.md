# Design Inspiration Sources

Seed sources saved from the initial Taptime design exploration. These are not
final approved screenshots. They are durable starting points for future visual
research and implementation work.

## Purpose

Use this file when you need to:

- resume the same design exploration later
- understand which sources were already considered
- know what each source is good for before collecting screenshots

## Saved Sources

### Product Hunt Home

- URL: <https://www.producthunt.com/>
- Role: trend radar for current product-brand tone
- Best for: headline tone, card density, launch-page polish
- Caution: do not use this alone as a mobile product UI reference

### Product Hunt Design & Creative

- URL: <https://www.producthunt.com/categories/design-creative>
- Role: discovery index for polished design-oriented products
- Best for: finding products worth opening and studying further
- Caution: treat it as a directory, not as the target reference itself

### Product Hunt UX Design Topic

- URL: <https://www.producthunt.com/topics/ux-design>
- Role: discovery index for UX-focused products
- Best for: finding newer products with clean hierarchy and interactions
- Caution: inspect the linked products directly before copying anything

### Mobbin

- URL: <https://mobbin.com/>
- Role: primary pattern source
- Best for: real-world app screens, flows, and common product decisions
- Caution: still curate aggressively; do not mix too many different products

### Land-book

- URL: <https://land-book.com/>
- Role: layout and atmosphere source
- Best for: visual rhythm, branded surfaces, hero composition, color feeling
- Caution: use for mood and layout ideas more than direct mobile UI structure

### Godly

- URL: <https://godly.website/>
- Role: anti-generic inspiration source
- Best for: bold identity, strong composition, less factory-made direction
- Caution: borrow sparingly to avoid overdesign

## Recommended Use Order

1. Start with Mobbin when the problem is a concrete app screen or flow.
2. Use Product Hunt to discover polished current products worth inspecting.
3. Use Land-book and Godly to push visual identity beyond default app patterns.
4. Save actual screenshots you want to preserve in
   `playgrounds/design-lab/references/images/`.
5. Register each approved screenshot in
   `playgrounds/design-lab/references/manifest.js`.

## Important Distinction

This file stores source links and selection rationale.

The actual implementation target should usually be one of:

- a screenshot in `playgrounds/design-lab/references/images/`
- a manifest entry with clear notes about what must match
- an approved token JSON exported from the design lab
